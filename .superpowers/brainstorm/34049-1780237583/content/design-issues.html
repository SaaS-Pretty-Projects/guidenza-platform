Design inconsistencies found across the codebase

These are the concrete issues. We'll fix all of them — the question below is about one key direction choice.

Inconsistency
Button shape split personality
Hero + Navbar use rounded-full. CTA section uses rounded-lg. Same action, different shape on the same page.
CTA.tsx: rounded-lg px-8 py-3.5
Should be: rounded-full px-8 py-3.5
Typo / broken
Hover colour class typo
Both Solution.tsx and Explore.tsx use hover:bg-white-[0.03] — invalid Tailwind. Should be hover:bg-white/[0.03]. Hover effect is silently broken.
Responsiveness
Footer 3-col grid breaks on tablet
Footer links use grid-cols-2 md:grid-cols-3 — on narrow tablets the third column orphans. Needs a proper responsive fallback.
Responsiveness
No search on mobile navbar
SearchBar is hidden md:flex — mobile users get no search at all. Should appear inside the mobile menu dropdown.
Inconsistency
Inner page headings don't match landing typography
Explore + Dashboard use plain text-4xl font-semibold. Landing uses tracking-[-2px] with serif italic accents. Inner pages feel generic by comparison.
Responsiveness
SearchChanged section top padding
pt-52 md:pt-64 — no tablet breakpoint. On 768–1024px viewports this creates an awkward gap between hero and this section.
Inconsistency
Explore page has no visual identity
Just a page title dropped into blank space. No overline label, no serif accent, no section structure — looks unfinished compared to everything else.
Polish
CoursePreview modal scroll on mobile
The modal doesn't account for mobile viewport height — on phones the enroll button can be hidden below the fold with no scroll indicator.

One direction call: how should inner pages (Explore, Dashboard) feel?

The landing is cinematic — full-bleed video, sticky scroll animations. Inner pages need a different register.
